UPC

Category: Program Dev-Compilers

Description

Cray provides a UPC compiler as one of several programming environments. The UPC compiler is a translator built with either PGI or GCC.

XTs have two kinds of nodes, but the UPC compilers only build executables to be run on the compute nodes. Thus

  • You should use the Cray wrappers cc, CC, and ftn
  • The wrappers will call the appropriate UPC compiler which will use the appropriate header files and link against the appropriate libraries.

The Cray wrappers cc, CC, and ftn will automatically link against the appropriate communication libraries and SciLib (-lsci). See the cc, CC, and ftn man pages for more information on their usage.

To see a man page on the UPC compilers with a full list of compiler options, first make sure either "PrgEnv-upc-pgi" or "PrgEnv-upc-gnu" module is loaded, and then do a man on "upcc".

Use

It is best to review the upcc man page.

To use the Cray wrappers to the UPC compilers, then either PrgEnv-upc-pgi or PrgEnv-upc-gnu module should be loaded: 

  module avail PrgEnv
  module swap PrgEnv-pgi PrgEnv-upc-gnu

To run the code, the executable must be launched with "upcrun" inside a batch job.
